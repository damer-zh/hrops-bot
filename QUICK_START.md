# 🚀 Быстрый Старт - Цифровой Пропуск

## Для Сотрудника 👤

### Получить пропуск

1. **Откройте приложение** HROps Bot
2. **Нажмите** кнопку `🪪` в верхнем правом углу
3. **Видите** красивую ID-карточку с QR-кодом
4. **QR обновляется** каждые 4 минуты (показывает оставшееся время внизу)
5. **Покажите** QR охраннику для проверки

## Для Охранника 👮

### Проверить пропуск (Способ 1: Камера - когда jsQR готовой)

1. **Откройте приложение** и перейдите на экран сканирования
2. **Нажмите** `Сканировать пропуск` или откройте `/?scan=true` в браузере
3. **Разрешьте** доступ к камере мобильного
4. **Наведите** камеру на QR-код сотрудника
5. **Автоматически** распознается и проверяется
6. **Видите** результат - ✅ валиден или ❌ невалиден

### Проверить пропуск (Способ 2: Ручной ввод - работает сейчас)

1. **Откройте приложение** на сканировании `/?scan=true`
2. **Нажмите** на экран или подождите диалог
3. **Вводите** токен (QR число/буквы) вручную
4. **Система** проверит токен
5. **Видите** результат

### Возможные Ошибки

| Ошибка | Причина | Решение |
|--------|---------|---------|
| **Token expired** | Пропуск старше 5 минут | Попросить новый пропуск |
| **Invalid signature** | Подделанный/повреждённый токен | Попросить новый пропуск |
| **Employee not found** | Сотрудник удалён из системы | Проверить в базе |
| **Employee inactive** | Сотрудник не имеет отдела | Сотруднику нужно завершить onboarding |

## Для Администратора 👨‍💼

### Конфигурация

**Бэкенд конфиг** - `src/HROpsBot.API/appsettings.json`:
```json
{
  "Pass": {
    "Secret": "CHANGE-THIS-TO-A-RANDOM-256-BIT-SECRET-IN-PRODUCTION"
  }
}
```

**В production** установить переменную окружения:
```bash
export HROPS_PASS_SECRET="your-generated-256-bit-secret"
```

### Генерировать Secret Key

**Linux/Mac:**
```bash
openssl rand -base64 32
```

**Windows PowerShell:**
```powershell
[Convert]::ToBase64String([System.Security.Cryptography.RandomNumberGenerator]::GetBytes(32))
```

### Мониторинг

**Проверить работу API:**

```bash
# Генерировать пропуск
curl http://localhost:5000/api/pass/generate/1

# Проверить пропуск
curl "http://localhost:5000/api/pass/verify?t=<token_from_above>"
```

**Логирование:**
- Все проверки логируются в application logs
- В будущем можно добавить PassAuditLog таблицу для детального отслеживания

### Развёртывание

**Docker:**
```bash
# Build
docker-compose build

# Run
docker-compose up -d

# Logs
docker-compose logs -f
```

## Безопасность

✅ **Что защищено:**
- HMAC-SHA256 подпись на каждом токене
- 5-минутный TTL (нельзя переиспользовать)
- Проверка временных атак (timing-safe comparison)
- Валидация сотрудника в базе перед выдачей пропуска

⚠️ **Требования production:**
- **HTTPS** обязательна (токен в URL)
- **Secret Key** в переменной окружения
- **Database backups** регулярные
- **Audit logs** для всех проверок

## Решение проблем

### Проблема: Камера не работает

**Решение:**
1. Проверьте разрешение на доступ к камере
2. Попробуйте другой браузер
3. Используйте fallback ввод токена вручную

### Проблема: "Token expired"

**Решение:**
1. Пропуск действует 5 минут
2. Попросить сотрудника сгенерировать новый пропуск (нажать 🪪 снова)

### Проблема: "Employee not found"

**Решение:**
1. Сотрудник удалён из системы
2. Добавить сотрудника обратно в базу данных

### Проблема: "Employee inactive"

**Решение:**
1. Сотруднику нужно завершить onboarding
2. Заполнить профиль (должность, отдел и т.д.)
